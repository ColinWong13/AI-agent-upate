一、核心概念与 Agent 原理

理解 AI Agent 所需的基础概念，为后续竞品分析提供统一的技术语言。

Token
大模型处理文本的最基本单位。0.75 个英文单词 ≈ 1 Token，1.5–2 个汉字 ≈ 1 Token。Tokenizer 通过 Unigram/BPE 算法完成编解码。
Context（上下文）
大模型每次推理时所接触到的信息总和，包括对话历史、用户问题、当前输出、工具列表和 System Prompt。主流大模型 Context Window 已达百万量级。
RAG（检索增强生成）
从输入 Context 中提取与用户提问最相关的片段进行推理，有效突破 Context 限制，提升效率与准确性。
Prompt（提示词）
User Prompt 说明具体任务（用户配置）；System Prompt 定义人设与行为规则（开发者配置）。
Function Calling
大模型调用外部工具的标准能力接口，是 Agent 与真实世界交互的核心机制。
MCP（模型上下文协议）
Model Context Protocol，各类工具接入不同 AI 平台的统一规范，大幅降低集成成本。
Agent
能够自主规划、调用一个或多个工具直至完成用户目标的系统。常见架构包括 ReAct、Plan and Execute 等。
Agent Skills
用户创建的、针对特定需求预先编写的规划/调用/输出/实现代码合集（提示词形式）。本质上实现 System Prompt + MCP 的功能。
工程范式
Agent Engineering / Prompt Engineering / Context Engineering / Harness Engineering 四大工程方向共同构成 Agent 开发体系。
二、主流 Agent 平台深度对比

按照行业派系划分为五大阵营：终端派、全能派、IDE派、开源派、中国力量。

2.1 行业派系全景：五大阵营

终端派

「终端是开发者的真实世界」

Claude Code
命令行优先、深度集成 Shell、计划模式（Plan Mode）、MCP 协议支持。逻辑严密性极强，原生支持 Unix 管道，SWE-bench 得分 81.4%。适合资深架构师、DevOps 工程师。
Aider
轻量级终端编码助手，Git 深度集成，支持多种大模型。适合命令行开发者和开源贡献者。

全能派

「一个模型解决所有问题」

OpenAI Codex
调试之王，拥有强大的代码压缩与推理能力，多语言支持，强大的代码理解能力。
Bolt.new
全栈 Web 应用生成、一键部署、WebContainer 技术。从提示词到部署的端到端体验，适合产品经理和创业者。

IDE 派

「IDE 是 AI 代理的舞台」

Cursor
AI 原生 IDE 金标准，以稳定性与 Composer 模式著称。代码库全索引，上下文管理强大。定价：免费 / $20 / $40 月。
Windsurf
Cascade 多文件编辑、Flows 自动化工作流，深度代码库理解。
GitHub Copilot
全球超 10 亿用户的微软生态，多模型支持（Claude 3.5 Sonnet、GPT-4o、o1）。定价：$10/$19 月。

开源派

「代码主权不容妥协」

Cline
开源 VS Code 扩展，MCP 协议支持，浏览器自动化能力。完全开源，隐私友好。
OpenCode
开源主权捍卫者，支持本地运行和 Docker 部署，保障数据安全。

中国力量

「全流程闭环，中国式敏捷」

Trae（字节跳动）
IDE 里的 Agent，具备 SOLO 模式的端到端开发能力。
Qoder（阿里巴巴）
工程标准与 Wiki 结合，解决 Spec Coding 上下文依赖。
CodeBuddy（腾讯）
产研一体化，实现从 PRD 到代码的 Spec 自动实现。

三、垂直领域 Agent 竞品分析

本节从 Data Agent、Finance Agent、Research Agent 三大垂直赛道展开竞品分析，并附技术路线选择建议。

技术路线说明：Agent 开发 vs Skills 开发

在进入各垂直领域分析前，需明确两条核心技术路线的差异：

Agent 开发
构建具备自主规划与工具调用能力的完整 Agent 系统，适合复杂、多步骤、跨工具的任务场景。
Skills 开发
针对特定高频需求预先编写标准化技能包，适合需要稳定、快速、可复用输出的场景。
3.1 Data Agent

市场现状：Data Agent 市场正从概念验证走向规模化应用。据 Gartner 预测，到 2026 年将有 65% 的企业部署 Data Agent；全球 Agentic AI 市场规模预计在 2032 年达到 932 亿美元。

市场竞品四大阵营

🏭 国内传统 BI 厂商转型

4.2 Skills 的三大核心优势
🚫 消除语法幻觉
通过预定义的结构化输出模板，强制 Agent 按照规范格式生成代码，避免因模型"创意发挥"导致的语法错误和不一致性。Skills 相当于给 Agent 装上了"语法护栏"。
🌍 增强环境感知
Skills 可以预先注入项目上下文（技术栈、代码规范、目录结构），让 Agent 在执行任务时始终"知道自己在哪里"，避免生成与项目环境不兼容的代码。
🧠 缓解遗忘效应
长对话中模型容易"忘记"早期约定的规范。Skills 通过每次触发时重新注入关键约束，确保 Agent 在整个任务周期内保持一致的行为模式，不因上下文压缩而退化。

4.4 原子化 TODO 管理：Skills 的最佳实践

原子化 TODO 管理是将复杂任务拆解为最小可执行单元的方法论，与 Skills 配合使用可实现 Agent 的自主闭环执行。核心理念：每个 TODO 只做一件事，且必须可独立验证。

原子化 TODO 四大原则
① 单一焦点
每个 TODO 只做一件事。避免"实现登录功能"这类模糊任务，应拆解为"创建 LoginForm 组件"、"实现 JWT 验证逻辑"等具体步骤。
② 自包含性
每个 TODO 包含执行所需的全部上下文：输入、输出、约束条件。Agent 无需查阅其他 TODO 即可独立完成当前任务。
③ 独立可测
每个 TODO 完成后可以独立验证结果，有明确的完成标准（如"单元测试通过"、"接口返回 200"），避免模糊的"完成"定义。
④ 限时完成
单个 TODO 的预估执行时间不超过 30 分钟。超过此阈值的任务应继续拆解，确保 Agent 在单次上下文窗口内完成并验证。

5.2 核心配置文件
📄 CLAUDE.md
项目级配置文件，放置在项目根目录。定义项目技术栈、代码规范、禁止行为、常用命令等。Claude Code 每次启动时自动读取，相当于项目专属的 System Prompt。
⚙️ settings.json
全局配置文件，位于 ~/.claude/settings.json。配置权限模式（autoApprove）、Hooks 钩子、MCP 服务器列表、模型选择等全局行为。
🛠️ Skills 目录
位于 ~/.claude/skills/，存放自定义 Skills 文件。每个 Skill 是一个 Markdown 文件，通过 /skill-name 斜杠命令触发，支持热重载（2.1 新特性）。
🔗 MCP 配置
在 settings.json 中配置 MCP 服务器，扩展 Claude Code 的工具能力。支持 Figma、Jira、Slack、GitHub、数据库等 30+ 官方 MCP 服务器，也可自定义开发。

六、总结与建议
6.1 市场格局判断

当前 AI Agent 市场呈现「平台层竞争激烈、垂直层机会分散」的格局：

🏗️ 平台层
OpenClaw、Hermes 等个人级 Agent 平台争夺「通知」与「辅助」入口，竞争焦点在生态丰富度、用户体验与成本控制。OpenAI、Google、Anthropic 三足鼎立，国内阿里云、腾讯、字节跳动紧随其后。
🎯 垂直层
Data/Finance/Research Agent 各自深耕行业 Know-How，竞争焦点在「准确率」、「可溯源」与「可信度」。垂直深耕仍有大量机会，尤其是金融、医疗、科研等高价值场景。
🔀 技术路线分化
Agent 开发适合复杂任务，Skills 开发适合高频标准化场景，两者将长期共存。超级个体需要「左手 Vibe（快，搞定界面与体验），右手 Spec（稳，搞定逻辑与架构）」。
6.2 技术趋势判断
🐝 多智能体协同
单一 Agent 向多 Agent 协作演进，「蜂群效应」将成为复杂任务的标准解法，TradingAgents 已验证可行性。
🚗 自治化升级
从 Copilot（辅助）到 Autopilot（自治），Agent 将承担更多端到端决策，信贷审批效率提升 99%。
✅ 可信与可解释
企业级场景中，幻觉消除与决策可溯源将成为核心竞争力，NL2SQL 准确率需达 95%+ 才可商用。
🛒 技能生态化
Skills 市场（类似 App Store）将成为 Agent 平台的重要护城河，ClawHub 已有上万种技能。
📍 边缘部署
隐私保护需求推动 Agent 向本地/边缘部署演进，轻量化模型价值凸显，MiroMind 单张 RTX 4090 可运行。
6.3 产品建议
建议方向具体行动
差异化定位避免与 OpenAI/Google 正面竞争，聚焦垂直场景或特定用户群体的深度需求
Skills 生态优先构建开放的 Skills 市场，通过社区驱动降低内容生产成本，提升平台粘性
可信 AI 为核在企业级产品中将「准确率」和「可溯源」作为核心卖点，而非单纯强调 AI 能力
轻量化部署提供本地部署选项，满足数据主权和隐私保护需求，尤其针对金融、医疗等敏感行业
平台选型建议追求广度与生态选 OpenClaw；追求深度与省心选 Hermes Agent；编程场景首选 Claude Code
