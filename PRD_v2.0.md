# AI 资讯合集平台 v2.0 产品需求文档

---

## 1. 项目背景

- **需求简介**：将现有静态 HTML 竞品调研报告升级为具备完整前后端、内容自动更新、用户可交互的动态 AI 资讯平台。
- **业务诉求**：解决当前静态页面内容不可更新、无用户体系、信息源依赖人工维护的痛点，实现资讯自动汇聚与个性化管理。

---

## 2. 业务流程简述

- 用户访问平台首页 → 浏览 AI Agent 竞品分析报告（现有内容结构化存储于数据库）。
- 用户在「学术板块」按顶刊、关键词、日期筛选论文列表，查看摘要与原文链接。
- 用户在「行业动态」按时间线浏览系统自动爬取的最新资讯。
- 用户注册/登录后 → 收藏资讯、添加笔记、管理个人数据库。
- 系统定时任务 → 自动爬取 arXiv / NeurIPS / ICLR 等顶刊论文、主流 AI 资讯源 → 写入数据库 → 前端动态渲染。

## 3. 详细功能说明

### 3.1 模块一：后端系统搭建（T0）

#### 3.1.1 技术选型
- **后端框架**：Python FastAPI
- **数据库**：PostgreSQL
- **缓存/队列**：Redis
- **定时任务**：APScheduler（轻量，适合单机部署）
- **前端方案**：保留现有纯 HTML/CSS 风格，使用 Jinja2 模板引擎动态渲染，按页面拆分为模板组件

> 选型理由：Python 生态在学术爬虫（arxiv API、scholarly）、内容抓取（BeautifulSoup、httpx）方面最为成熟；FastAPI 原生异步支持可高效处理并发爬取与 API 请求；PostgreSQL 全文检索能力满足论文/资讯搜索需求。

#### 3.1.4 定时任务配置

- **arXiv 爬取**：每日 08:00（北京时间）执行，获取前一日新论文。
- **NeurIPS 爬取**：每周一 09:00 执行。
- **ICLR 爬取**：每周一 10:00 执行。
- **行业动态爬取**：每 6 小时执行一次。
- **重复检查**：以 url 字段为唯一键，插入时自动跳过已存在记录。
- **失败重试**：单次爬取失败后，间隔 30 分钟重试一次；连续 3 次失败则记录日志并跳过本次周期。

### 3.2 模块二：学术论文板块（T1）

- **位置**：导航栏 - 学术板块
- **目标**：自动汇聚顶刊最新论文，支持筛选与检索

#### 3.2.1 论文列表页

**顶部筛选栏**
- 顶刊来源：多选框，选项为「arXiv」「NeurIPS」「ICLR」「ICML」「ACL」，默认全选。
- 关键词输入框：文本输入，最大 100 字符，按回车触发搜索。
- 日期范围：起止日期选择器，默认近 30 天。
- 排序方式：单选，「最新发布」「标题 A-Z」两选项，默认最新发布。

**列表展示**
- 每页 20 条，底部为分页器（上一页 / 页码 / 下一页）。
- 每条论文卡片显示：
  - 标题（最大 2 行，超出截断）
  - 作者列表（展示前 3 位 +「等」，悬停显示全部）
  - 来源标签（颜色区分：arXiv 蓝色、NeurIPS 绿色、ICLR 橙色、ICML 紫色、ACL 红色）
  - 摘要（最大 3 行，超出截断）
  - 发布日期
- 点击标题或卡片 → 跳转论文详情页。
- 论文标题旁有「收藏」图标（☆），登录用户可点击收藏（★已收藏）。

**空状态**
- 无数据时显示：「暂无符合条件的论文」。
- 加载中显示骨架屏（灰色占位卡片 × 6）。

**异常处理**
- 接口超时或返回错误时，卡片区域显示：「数据加载失败，点击重试」按钮。
- 点击重试 → 重新请求接口。

#### 3.2.2 论文详情页

- 展示完整标题、全部作者（逗号分隔）、来源、发布日期、PDF 链接。
- 摘要全文展示。
- 底部操作区：
  - 「查看原文」按钮 → 新标签页打开论文链接。
  - 「下载 PDF」按钮 → 新标签页打开 PDF 链接。
  - 「收藏」按钮 → 已登录用户可点击切换收藏状态。
  - 「添加笔记」按钮 → 已登录用户点击后展开笔记文本区。
- 笔记文本区：
  - 多行文本输入框，最大 5000 字符。
  - 字符计数器：实时显示「{N}/5000」。
  - 「保存笔记」按钮 → 点击保存，成功提示「笔记已保存」，失败提示「保存失败，请重试」。

#### 3.2.3 爬取来源与策略

- **arXiv**：通过官方 API（`http://export.arxiv.org/api/query`）获取，支持 cs.AI / cs.CL / cs.LG / stat.ML 四个分类。
- **NeurIPS**：爬取 Proceedings 页面（`https://papers.nips.cc`），按年份筛选最新会议论文。
- **ICLR**：爬取 OpenReview API，获取最新 ICLR 会议论文。
- **ICML**：爬取 PMLR（Proceedings of Machine Learning Research）页面。
- **ACL**：爬取 ACL Anthology 页面。
- **去重规则**：以 title + published_date 组合为唯一键，避免重复入库。
- **爬取频率限制**：单源两次请求间隔 ≥ 3 秒，避免触发反爬。
- **版权合规**：仅爬取公开元数据（标题、作者、摘要、链接），不下载和存储全文 PDF。

### 3.3 模块三：行业动态自动爬取（T1）

- **位置**：导航栏 - 行业动态
- **目标**：替代现有静态外链，自动从多源汇聚 AI 行业资讯

#### 3.3.1 动态列表页

**顶部筛选栏**
- 分类筛选：多选框，选项为「大模型」「Agent」「开源工具」「融资收购」「政策法规」，默认全选。
- 时间范围：单选，「近 24 小时」「近 7 天」「近 30 天」，默认近 7 天。
- 搜索框：文本输入，最大 100 字符，按回车触发。

**列表展示**
- 每页 20 条，底部为分页器。
- 每条动态卡片显示：
  - 来源名称（如「机器之心」「量子位」「Ars Technica」）
  - 标题
  - 摘要（最大 3 行）
  - 发布时间（24 小时内显示相对时间如「3 小时前」，超过 24 小时显示绝对日期）
  - 分类标签
- 点击标题 → 新标签页打开原文链接。
- 已登录用户可点击「收藏」图标。

**空状态**
- 无数据时：「暂无符合条件的动态」。
- 加载中：骨架屏。

**异常处理**
- 接口异常时：列表区域显示「加载失败，点击重试」。

#### 3.3.2 爬取来源

| 来源 | 类型 | 说明 |
|------|------|------|
| 机器之心 | 中文 RSS | 大模型与 Agent 深度报道 |
| 量子位 | 中文 RSS | AI 行业前沿资讯 |
| 新智元 | 中文 RSS | AI 技术与产业动态 |
| MIT Technology Review | 英文 RSS | AI 学术与产业交叉报道 |
| Ars Technica (AI) | 英文 RSS | 技术深度解读 |
| The Verge (AI) | 英文 RSS | 消费级 AI 产品动态 |

#### 3.3.3 爬取策略

- 每个来源通过 RSS 订阅接口获取，解析 XML 提取标题、摘要、链接、时间。
- 以 url 为唯一键去重。
- 中文来源每 3 小时抓取一次，英文来源每 6 小时抓取一次。
- 单源失败不影响其他来源，失败记录日志。

### 3.4 模块四：UI 与内容优化（T2）

- **位置**：全局
- **目标**：提升视觉一致性、阅读体验和移动端适配

#### 3.4.1 导航栏优化

- 导航项从原有 8 个调整为：「首页」「竞品报告」「学术板块」「行业动态」。
- 用户未登录时，导航栏右侧显示「注册」「登录」按钮。
- 用户已登录时，导航栏右侧显示用户昵称 + 下拉菜单（「我的收藏」「我的笔记」「退出登录」）。
- 当前所在页面对应的导航项高亮（active 态）。
- 移动端（宽度 ≤ 768px）：导航栏折叠为汉堡菜单，点击展开下拉导航列表。

#### 3.4.2 首页重构

- 保留 Hero 区域（标题 + 简介 + 标签），但数据从数据库动态加载。
- Hero 下方为「今日快讯」区域：展示最近 24 小时内爬取的热门动态（前 5 条）。
- 「热门论文」区域：展示最近 7 天内收藏数最多的前 6 篇论文。
- 「竞品报告速览」区域：4 个卡片分别链接到核心概念、平台对比、垂直领域、Skills 指南。

#### 3.4.3 响应式适配

- 断点：768px（平板）、480px（手机）。
- 桌面端（>768px）：多列网格布局，导航栏横向展开。
- 平板端（481px-768px）：两列布局，导航栏仍横向展开但字体缩小。
- 手机端（≤480px）：单列布局，汉堡菜单，表格改为横向滚动。
- 所有文字使用相对单位（rem），图片使用 max-width: 100% 自适应。

#### 3.4.4 内容优化

- 原有 HTML 中所有静态文本迁移至数据库 report_sections 表的 content_json 字段。
- 后台管理页面提供 Markdown 编辑器，支持编辑各章节内容并实时预览。
- 竞品报告中的表格数据支持筛选与排序（前端 JS 实现）。
- 新增「数据更新日志」页面，展示各数据源最近更新时间与条目数。

#### 3.4.5 暗色模式

- 导航栏右侧增加「浅色/暗色」切换按钮（🌙/☀️图标）。
- 首次访问跟随系统偏好（prefers-color-scheme）。
- 用户手动切换后 → 存储到 localStorage，下次访问读取本地偏好。
- 暗色模式 CSS 变量覆盖：
  - 背景色：#1a1a2e → 原 #f4f6fb
  - 卡片背景：#2a2a3e → 原 #fff
  - 文字色：#e0e0e0 → 原 #1a1a2e

### 3.5 模块五：用户数据库管理系统（T2）

- **位置**：导航栏 - 用户昵称下拉 - 「我的收藏」「我的笔记」
- **目标**：让用户管理自己的资讯收藏与笔记库

#### 3.5.1 注册页

- **位置**：导航栏 - 注册
- 表单字段：
  - 昵称：文本输入，2-20 字符，实时校验（失焦时提示「昵称需 2-20 个字符」）。
  - 邮箱：文本输入，失焦时校验邮箱格式（正则匹配），不合法时提示「请输入有效邮箱」。
  - 密码：密码输入框，最少 8 字符，失焦时校验。
  - 确认密码：密码输入框，失焦时校验是否与密码一致，不一致时提示「两次密码输入不一致」。
- 点击「注册」按钮：
  - 表单校验全部通过后，按钮显示加载状态。
  - 成功 → 自动登录，跳转首页，顶部绿色提示「注册成功」。
  - 失败 → 按钮恢复，顶部红色提示具体错误原因（如「该邮箱已被注册」）。

#### 3.5.2 登录页

- **位置**：导航栏 - 登录
- 表单字段：
  - 邮箱：文本输入。
  - 密码：密码输入框。
- 点击「登录」按钮：
  - 按钮显示加载状态。
  - 成功 → 跳转上一页面或首页，顶部绿色提示「登录成功」。
  - 失败 → 按钮恢复，顶部红色提示「邮箱或密码错误」。
- 登录态有效期：JWT Token 7 天过期，过期后需重新登录。

#### 3.5.3 我的收藏

- **位置**：导航栏 - 用户下拉 - 我的收藏
- 列表展示已收藏的论文和动态，按收藏时间倒序。
- Tab 切换：「论文」（默认）「动态」。
- 每条记录可点击跳转详情，右侧有「取消收藏」按钮。
- 「取消收藏」按钮点击 → 确认弹窗「确定取消收藏？」，确认后从列表移除。
- 空状态：「暂无收藏，去{学术板块/行业动态}逛逛吧」。

#### 3.5.4 我的笔记

- **位置**：导航栏 - 用户下拉 - 我的笔记
- 列表展示所有笔记，按更新时间倒序。
- 每条显示：关联的论文/动态标题、笔记摘要（前 50 字符）、更新时间。
- 点击进入编辑模式：展示完整笔记内容，支持修改并保存。
- 「删除笔记」按钮 → 确认弹窗「确定删除该笔记？」，确认后删除。
- 空状态：「暂无笔记」。

### 5.2 分支策略

- `main`：生产分支，仅通过 PR 合并。
- `develop`：开发主线分支。
- `feature/v2.X.X-xxx`：功能分支，从 develop 切出，完成后 PR 合并回 develop。
- `hotfix/xxx`：线上紧急修复分支。

### 5.3 提交规范

- 格式：`<type>: <subject>`
- type 取值：`feat`（新功能）、`fix`（修复）、`refactor`（重构）、`docs`（文档）、`test`（测试）、`chore`（工程配置）

## 6. 测试用例概要

### 6.1 爬虫模块

| 编号 | 场景 | 预期结果 |
|------|------|---------|
| TC-C01 | arXiv 爬取正常返回数据 | 数据入库，字段完整 |
| TC-C02 | 爬取源不可达（超时） | 记录错误日志，30 分钟后重试 |
| TC-C03 | 重复条目入库 | 自动跳过，不产生重复记录 |
| TC-C04 | 连续 3 次失败 | 跳过本次周期，记录告警日志 |
| TC-C05 | 空结果（某源当日无新内容） | 正常结束，不报错 |

### 6.2 API 接口

| 编号 | 场景 | 预期结果 |
|------|------|---------|
| TC-A01 | GET /api/papers 无筛选条件 | 返回分页列表，按时间倒序 |
| TC-A02 | GET /api/papers?keyword=transformer | 返回标题/摘要含 transformer 的论文 |
| TC-A03 | GET /api/papers?source=arxiv&date_from=2026-04-01 | 返回符合条件的筛选结果 |
| TC-A04 | POST /api/user/register 重复邮箱 | 返回 409，提示「该邮箱已被注册」 |
| TC-A05 | POST /api/user/login 错误密码 | 返回 401，提示「邮箱或密码错误」 |
| TC-A06 | 无 Token 访问收藏接口 | 返回 401，提示「请先登录」 |
| TC-A07 | Token 过期访问接口 | 返回 401，提示「登录已过期」 |

### 6.3 前端页面

| 编号 | 场景 | 预期结果 |
|------|------|---------|
| TC-F01 | 桌面端访问首页 | 正常展示 Hero + 快讯 + 热门论文 + 报告速览 |
| TC-F02 | 手机端（375px 宽度）访问 | 单列布局，汉堡菜单可用 |
| TC-F03 | 切换暗色模式 | 全局配色切换，刷新后保持偏好 |
| TC-F04 | 未登录点击收藏 | 弹出提示「请先登录」或跳转登录页 |
| TC-F05 | 已登录点击收藏 | 图标切换为★已收藏状态 |
| TC-F06 | 论文列表空数据 | 显示空状态提示 |
| TC-F07 | 接口请求超时 | 显示「加载失败，点击重试」 |
| TC-F08 | 笔记输入 5000 字以上 | 阻止输入，显示「5000/5000」红色提示 |
